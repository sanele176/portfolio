Linux Automation with Ansible

Ansible is an open-source IT automation tool that can configure systems, deploy software, and orchestrate more advanced IT tasks such as continuous deployments or zero downtime rolling updates. It’s written in Python and uses a human-readable data serialization language YAML for playbook writing. Ansible works by connecting to your nodes and pushing out small programs, called “Ansible modules” to them. These programs are written to be resource models of the desired state of the system. Ansible then executes these modules over SSH by default, and removes them when finished.

Firmware Upgrade on HPE Servers

Objectives

  • Improving functionality and features on your devices
  • Applying fixes to any performance issues
  • Applying device drivers to enhance performance

Steps

  1. Obtain Valid SPP ISO image from HPE Software center
  2. Obtain Valid ILO credentials
  3. Use windows server to connect to ILO portal of target device
  4. Power down device
  5. Open console session to target device and attach ISO image
  6. Power up device and boot into HPE SPP
  7. Select interactive mode, review avalable updates and apply
  8. Reboot device
  9. To only upgrade firmware obtain .bin file and flush firmware

File share to sharepoint online Migration

Aproach 1 : Using Sharepoint Migration manager

  1. Account with read permissions on the shares to be migrated
  2. Sharepoint online admin account
  3. On sharepoint admin centre, navigate to migration
  4. Under agents download,install and configure an agent
  5. On the agent configuration Ensure to configure the source file share
  6. Ensure agent status after configuration is Enabled
  7. Create a migration task, ensure correct sharepoint destination site
  8. Schedule or run migration task
  9. Ensure migration server has sufficient storage space


Aproach 2 : Using Sharepoint Migration Tool

  1. Account with read permissions on the shares to be migrated
  2. Sharepoint online admin account
  3. Download Sharepoint Migration Tool
  4. Launch and provide windows credentials with access to file shares and credentials for sharepoint admin centre
  5. Create a migration task by providing source file share and destination on sharepoint online
  6. Schedule or run migration task
  7. Ensure migration server has sufficient storage space
